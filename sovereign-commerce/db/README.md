# Database Schema for Sovereign Commerce Platform

This directory contains database schema files and migration scripts for the Sovereign Commerce platform designed for diaspora funders.

## Files

- `schema.sql` - Complete database schema with all tables
- `migrations/` - Database migration scripts
- `seeds/` - Sample data for development and testing
- `sovereign_commerce.db` - SQLite database file

## Schema Overview

### Core Tables

#### Users
- Multi-tier role system (funder, flamekeeper, custodian)
- Sigil-based sovereign recognition
- Authority levels and ceremonial status
- Diaspora community integration

#### Products
- Sacred offerings catalog
- Ceremonial tier classification
- Metadata for enhanced descriptions
- Category-based organization

#### Orders & Order Items
- Complete purchase workflow tracking
- Ceremonial seals for each order
- Price preservation at time of purchase
- Detailed item-level records

#### Audit Trail
- Complete transaction logging
- Ceremonial witness tracking
- Old/new value preservation
- Governance and accountability

#### Sigils
- Sovereign recognition system
- Authority-granted permissions
- Validity periods and power levels
- Active status management

#### Ceremonial Sessions
- Enhanced session tracking
- Ceremonial context preservation
- Security and user agent logging
- Multi-tier session types

## Design Principles

### Sovereignty
- User autonomy and self-determination
- Ceremonial recognition of authority
- Diaspora community values

### Governance
- Audit trails for all actions
- Multi-tier authority systems
- Accountability and transparency

### Ceremony
- Sacred elements in commerce
- Blessed transactions
- Meaningful engagement patterns

### Security
- Role-based access control
- Session management
- Token-based authentication

## Usage

The database is designed to support:
- Diaspora funder commerce needs
- Ceremonial transaction processing
- Sovereign recognition systems
- Multi-tier authority structures
- Complete audit and governance
